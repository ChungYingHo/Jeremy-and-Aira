## 淘汰與搜尋法概論

淘汰與搜尋法（Prune and Search）的定義：

> 給定一個搜尋空間，每一步利用某種比較或測試**刪除其中一部分**，只在剩下的部分繼續搜尋，直到找到答案或剩下單一元素。

核心特徵：

- 必須存在一個**判斷規則**，能在低成本（常數時間或線性時間）內決定哪一部分必然不含答案
- 每一輪刪除的比例越大、剩餘規模收斂越快，整體複雜度越好
- 被刪除的部分**不再回頭檢查**

如果每一輪能將規模從 $n$ 縮成 $n / c$（$c > 1$ 為固定常數）：

- 判斷成本為 $O(1)$：$T(n) = T(n/c) + O(1) = O(\log_c n)$
- 判斷成本為 $O(n)$：$T(n) = T(n/c) + O(n) = O(n)$

底數 $c$ 由「一次比較能產生多少種互斥結果」決定。

## 偽幣問題

問題定義：

- 共有 $n$ 枚硬幣，外觀相同
- 其中**恰有一枚是偽幣，已知比真幣輕**
- 工具：一個無砝碼天平
- 目標：找出偽幣，最小化秤量次數

### 演算法步驟

1. 將 $n$ 枚硬幣分成三組，大小盡量相等：左組、右組、未上秤組
2. 將左組與右組放上天平比較
3. 根據結果縮小搜尋範圍：
   - 左輕：偽幣在左組
   - 右輕：偽幣在右組
   - 平衡：偽幣在未上秤組
4. 對被選中的那一組遞迴重複步驟 1–3，直到剩下一枚

### 例題：9 枚硬幣

設 9 枚硬幣編號 0–8，偽幣為 5。

| 步驟 | 左盤 | 右盤 | 未上秤 | 結果 | 候選 |
| :---: | :---: | :---: | :---: | :--- | :--- |
| 1 | {0,1,2} | {3,4,5} | {6,7,8} | 右盤輕 | {3,4,5} |
| 2 | {3} | {4} | {5} | 平衡 | {5} |

兩次秤量找出偽幣。

### 為什麼分三組：底數的選擇

天平一次比較可以產生**三種互斥結果**（左輕、右輕、平衡），這代表一次比較能將搜尋空間切成三等分，每次只保留其中一份：

$$T(n) = T(n/3) + O(1) = O(\log_3 n)$$

若改用只能回答「是 / 否」兩種結果的工具（例如真假鈔辨識筆，每次只能告訴你「這枚是不是偽幣」），則一次比較只能切成兩等分：

$$T(n) = T(n/2) + O(1) = O(\log_2 n)$$

通則：

- 比較工具能產生 $k$ 種互斥結果 → 每次淘汰可將搜尋空間縮為 $1/k$
- 時間複雜度為 $O(\log_k n)$
- 底數 $k$ 由**比較工具的解析度**決定，不是隨意選擇

| 比較工具 | 一次結果數 | 每輪保留 | 複雜度 |
| :--- | :---: | :---: | :---: |
| 真假辨識筆（是/否） | 2 | $1/2$ | $O(\log_2 n)$ |
| 天平（左/右/平衡） | 3 | $1/3$ | $O(\log_3 n)$ |

## 晶片測試問題

問題定義：

- 共有 $n$ 個晶片，其中好晶片的數量嚴格大於 $n / 2$
- 每次測試取兩個晶片互測，回報結果如下：
  - 兩好：雙方都回報「對方是好的」
  - 一好一壞：好的誠實回報；壞的回報任意
  - 兩壞：雙方回報任意
- 目標：以最少測試次數找出**任一**好晶片

### 演算法步驟

1. 將晶片兩兩配對互測
2. 對每一對的回報：
   - 兩者皆回報「對方好」→ **保留其中一個**，另一個丟棄
   - 任一回報「對方壞」→ **兩個都丟棄**
3. 若 $n$ 為奇數，落單那一片暫時保留
4. 對保留集合遞迴執行步驟 1–3，直到剩下一片

### 正確性簡證

關鍵觀察：在「兩者都回報好」的配對中，兩片必為**同一類**（同好或同壞），不可能一好一壞——因為若一好一壞，好的會誠實回報壞的為壞。

設原本好晶片數為 $g > n/2$，壞晶片為 $b < n/2$。一輪過後：

- 「兩好」配對：保留一片好，丟一片好 → 好晶片數量減半
- 「兩壞」配對：保留一片壞或兩片皆丟（依結果）→ 壞晶片至少減半
- 「一好一壞」配對：必為「對方壞」，兩片都丟 → 各丟掉一好一壞

過濾後好晶片數量仍嚴格大於壞晶片數量，遞迴條件保持成立。

### 複雜度

每輪規模約減半，比較成本 $O(n)$：

$$T(n) = T(n/2) + O(n) = O(n)$$

線性測試次數即可找到一片好晶片。

## 二元搜尋

問題定義：

- 已排序陣列 $a[0..n-1]$
- 給定目標值 $x$
- 找出 $x$ 在陣列中的索引，或回報不存在

### 演算法步驟

1. 設 $\text{lo} = 0$，$\text{hi} = n - 1$
2. 取中點 $\text{mid} = \lfloor (\text{lo} + \text{hi}) / 2 \rfloor$
3. 比較 $a[\text{mid}]$ 與 $x$：
   - 相等：回傳 $\text{mid}$
   - $a[\text{mid}] < x$：$x$ 必在右半 → $\text{lo} \leftarrow \text{mid} + 1$
   - $a[\text{mid}] > x$：$x$ 必在左半 → $\text{hi} \leftarrow \text{mid} - 1$
4. 重複直到 $\text{lo} > \text{hi}$ 或找到 $x$

### 例題

陣列 $[3, 7, 12, 18, 25, 31, 42, 56, 64, 78, 85, 91]$，搜尋 $x = 56$。

| 步驟 | lo | hi | mid | $a[\text{mid}]$ | 動作 |
| :---: | :---: | :---: | :---: | :---: | :--- |
| 1 | 0 | 11 | 5 | 31 | $31 < 56$，淘汰左半 |
| 2 | 6 | 11 | 8 | 64 | $64 > 56$，淘汰右半 |
| 3 | 6 | 7 | 6 | 42 | $42 < 56$，淘汰左半 |
| 4 | 7 | 7 | 7 | 56 | 命中 |

<BinarySearchDiagram client:only="svelte" />

### 複雜度

每輪比較一次（$O(1)$），規模減半：

$$T(n) = T(n/2) + O(1) = O(\log n)$$

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

## 選出第 k 小的元素

問題定義：

- 給定 $n$ 個未排序的元素
- 找出第 $k$ 小的元素（$1 \leq k \leq n$）

淘汰與搜尋法的核心：選一個 pivot 將陣列分成「小於 pivot」「等於 pivot」「大於 pivot」三部分，根據第 $k$ 小落在哪一部分，**只遞迴其中一部分**，其餘丟棄。

兩種變體在於 pivot 的選擇方式——一個保證最差情況、一個保證期望情況。

### 最差情況最佳化：Median of Medians（BFPRT）

由 Blum、Floyd、Pratt、Rivest、Tarjan 於 1973 年提出，能在最差情況下達到 $O(n)$。

#### 步驟

1. 將 $n$ 個元素分成 $\lceil n/5 \rceil$ 組，每組 5 個（最後一組可能不滿）
2. 每組內排序，取中位數，得到 $\lceil n/5 \rceil$ 個中位數
3. **遞迴呼叫 Median of Medians**，找出這些中位數的中位數 $m$
4. 以 $m$ 為 pivot 分割原陣列為 $L$（小於 $m$）、$E$（等於 $m$）、$G$（大於 $m$）
5. 根據 $k$ 的位置：
   - $k \leq |L|$：在 $L$ 中遞迴找第 $k$ 小
   - $|L| < k \leq |L| + |E|$：答案就是 $m$
   - 否則：在 $G$ 中遞迴找第 $(k - |L| - |E|)$ 小

<MedianOfMediansDiagram client:only="svelte" />

#### 為什麼分組大小是 5

關鍵在於 pivot $m$ 必須保證能淘汰**固定比例**的元素。設總共 $\lceil n/5 \rceil$ 個中位數，$m$ 是其中位數，因此：

- 至少有 $\lceil \frac{1}{2} \cdot \lceil n/5 \rceil \rceil$ 個組別中位數 $\leq m$
- 在這些組裡，每組有 3 個元素 $\leq m$（組內排序後位於前 3 名）
- 因此至少有 $3 \cdot \frac{1}{2} \cdot \frac{n}{5} = \frac{3n}{10}$ 個元素 $\leq m$

對稱地，至少有 $\frac{3n}{10}$ 個元素 $\geq m$。所以每次遞迴最壞情況最多剩下：

$$n - \frac{3n}{10} = \frac{7n}{10}$$

遞迴式：

$$T(n) = T(n/5) + T(7n/10) + O(n)$$

由於 $\frac{1}{5} + \frac{7}{10} = \frac{9}{10} < 1$，此遞迴式解為 $T(n) = O(n)$。

#### 為什麼不用 3 或 7

| 組大小 | 每組保證 $\leq m$ 的個數 | 最壞剩餘 | 遞迴式 | 是否線性 |
| :---: | :---: | :---: | :--- | :---: |
| 3 | 2 | $2n/3$ | $T(n) = T(n/3) + T(2n/3) + O(n)$ | 否（$1/3 + 2/3 = 1$） |
| 5 | 3 | $7n/10$ | $T(n) = T(n/5) + T(7n/10) + O(n)$ | 是（和為 $9/10$） |
| 7 | 4 | $5n/7$ | $T(n) = T(n/7) + T(5n/7) + O(n)$ | 是（和為 $6/7$） |

組大小為 3 時遞迴式總和為 1，無法收斂為線性；組大小為 5 是「最小可線性」的選擇，組大小為 7 雖也線性但常數更大，分組與排序成本增加。**5 是最優平衡點**。

#### 例題：分組過程

設 25 個元素：

```
[12, 3, 17, 8, 5,  21, 14, 9, 2, 19,  7, 25, 11, 16, 4,  22, 1, 18, 10, 13,  6, 23, 15, 20, 24]
```

分成 5 組後組內排序：

| 組 | 內容（排序後） | 中位數 |
| :---: | :--- | :---: |
| G1 | 3, 5, **8**, 12, 17 | 8 |
| G2 | 2, 9, **14**, 19, 21 | 14 |
| G3 | 4, 7, **11**, 16, 25 | 11 |
| G4 | 1, 10, **13**, 18, 22 | 13 |
| G5 | 6, 15, **20**, 23, 24 | 20 |

中位數集合 $\{8, 14, 11, 13, 20\}$ 排序後為 $\{8, 11, 13, 14, 20\}$，其中位數為 $13$ → pivot。

以 $m = 13$ 分割原陣列，根據 $k$ 落在哪一邊遞迴。

### 平均情況最佳化：Randomized Select

#### 步驟

1. 從陣列中**隨機**選一個元素作為 pivot $p$
2. 分割為 $L$（$< p$）、$E$（$= p$）、$G$（$> p$）
3. 根據 $k$ 的位置遞迴 $L$ 或 $G$，或直接回傳 $p$

```python
import random

def randomized_select(arr, k):
    if len(arr) == 1:
        return arr[0]
    pivot = random.choice(arr)
    L = [x for x in arr if x < pivot]
    E = [x for x in arr if x == pivot]
    G = [x for x in arr if x > pivot]
    if k <= len(L):
        return randomized_select(L, k)
    elif k <= len(L) + len(E):
        return pivot
    else:
        return randomized_select(G, k - len(L) - len(E))
```

#### 期望複雜度推導

設 $T(n)$ 為期望比較次數。隨機 pivot 落在第 $i$ 名的機率均等為 $1/n$，最差情況下遞迴規模為 $\max(i - 1, n - i)$。可證明期望剩餘規模不超過 $3n/4$，所以：

$$T(n) \leq T(3n/4) + O(n)$$

展開為幾何級數：

$$T(n) \leq c \cdot n \cdot \left(1 + \frac{3}{4} + \left(\frac{3}{4}\right)^2 + \cdots\right) = 4cn = O(n)$$

#### 最差情況

每次 pivot 都恰為當前最大或最小值時：

$$T(n) = T(n - 1) + O(n) = O(n^2)$$

但這需要連續 $n$ 次極差選擇，機率為 $1/n!$，實務上幾乎不會發生。

### 兩種變體比較

| | Median of Medians | Randomized Select |
| :---: | :---: | :---: |
| pivot 選擇 | 中位數的中位數 | 隨機 |
| 期望複雜度 | $O(n)$ | $O(n)$ |
| 最差複雜度 | $O(n)$ | $O(n^2)$ |
| 常數 | 大 | 小 |
| 實務使用 | 少 | 多 |

## 打包問題（Bin Packing）

問題定義：

- 給定 $n$ 個物品，大小分別為 $s_1, s_2, \ldots, s_n$，皆滿足 $0 < s_i \leq C$
- 給定箱子容量 $C$
- 目標：使用**最少數量**的箱子裝下所有物品

Bin Packing 為 **NP-Hard** 問題，無已知多項式時間最佳解。實務上以**近似演算法**為主，淘汰與搜尋法不是 Bin Packing 的主力解法，但其中的**剪枝精神**與淘汰與搜尋法相通——分支限界（Branch and Bound）會剪掉「已知不可能優於目前最佳解」的子樹。

### WG 與 WE：浪費的兩種來源

要分析近似演算法的好壞，需要量化「裝箱不夠緊」造成的浪費：

- **WG（Wasted Group / Wasted Bin）**：一個箱子被關閉（不再放入物品）時，**剩餘的未使用容量**。例如容量 $C = 10$，箱子裝了 7 後關閉，則該箱貢獻 $\text{WG} = 3$。
- **WE（Wasted Element）**：一個物品因為**現有所有箱子都塞不下**，被迫開新箱來容納。這個物品本身就是一次「淘汰失敗」事件——它被現有箱子全部拒絕。

兩者關係：

- 總空間使用 = 物品總大小 + 所有 WG 的總和
- 開的箱子數 $\geq \lceil$ 物品總大小 $/ C \rceil$，超出的部分由 WG 累積決定
- 每出現一個 WE，至少多開一個箱子，並可能在原本箱子留下新的 WG

### 不同近似策略對 WG / WE 的影響

| 策略 | 機制 | WG 趨勢 | WE 趨勢 |
| :--- | :--- | :--- | :--- |
| Next Fit | 只看最後一個箱子，塞不下就開新箱 | 高（前面箱子永遠不再使用） | 高 |
| First Fit | 從第一個箱子掃，第一個塞得下就放 | 中（會回頭利用） | 中 |
| Best Fit | 選擇剩餘空間最少且仍能容納的箱子 | 較低（剩餘空間集中於少數箱） | 中 |
| First Fit Decreasing | 先將物品由大到小排序，再 First Fit | 低（小物件用來填縫） | 低 |

直覺上：先放大的物品，剩下的縫隙才有機會被小物品填滿——這就是 FFD 比 FF 更好的原因。

### 用 WG / WE 推導近似比上界

設 $\text{OPT}$ 為最佳解的箱子數，演算法產生的箱子數為 $A$。則：

$$A \cdot C = \sum_{i=1}^{n} s_i + \sum_{\text{箱子}} \text{WG}$$

由於 $\text{OPT} \cdot C \geq \sum s_i$，可得：

$$A \leq \text{OPT} + \frac{\sum \text{WG}}{C}$$

近似比的證明就是在估算 $\sum \text{WG}$ 的上界。經典結果：

- **First Fit**：$A \leq 1.7 \cdot \text{OPT} + 2$
- **First Fit Decreasing**：$A \leq \frac{11}{9} \cdot \text{OPT} + \frac{6}{9}$

WE 的角色：每次發生 WE 就代表某個物品**無法放入任何既有箱子**——這意味著所有既有箱子的剩餘空間都小於該物品大小。透過此性質可以證明 First Fit 中「剩餘空間 $< 1/2$ 的箱子數量」具有特定上界，再結合 WG 累積導出近似比。

### 例題：First Fit Decreasing

物品：$[0.5, 0.7, 0.5, 0.2, 0.4, 0.2, 0.3, 0.1]$，箱容量 $C = 1.0$

排序後（由大到小）：$[0.7, 0.5, 0.5, 0.4, 0.3, 0.2, 0.2, 0.1]$

| 步驟 | 物品 | 動作 | 箱況（剩餘容量） | WG 變化 | WE |
| :---: | :---: | :--- | :--- | :---: | :---: |
| 1 | 0.7 | 開 Bin1 | Bin1: 0.3 | — | — |
| 2 | 0.5 | Bin1 塞不下（0.3），開 Bin2 | Bin1: 0.3, Bin2: 0.5 | — | +1 |
| 3 | 0.5 | Bin1 塞不下，Bin2 剛好 | Bin1: 0.3, Bin2: 0.0 | — | — |
| 4 | 0.4 | Bin1 塞不下，Bin2 塞不下，開 Bin3 | Bin1: 0.3, Bin2: 0.0, Bin3: 0.6 | — | +1 |
| 5 | 0.3 | Bin1 剛好 | Bin1: 0.0, Bin2: 0.0, Bin3: 0.6 | — | — |
| 6 | 0.2 | Bin3 可塞 | Bin3: 0.4 | — | — |
| 7 | 0.2 | Bin3 可塞 | Bin3: 0.2 | — | — |
| 8 | 0.1 | Bin3 可塞 | Bin3: 0.1 | — | — |

最終結果：

- 使用 3 個箱子
- Bin1 WG = 0，Bin2 WG = 0，Bin3 WG = 0.1
- 總 WG = 0.1，總 WE 事件 = 2（步驟 2、4）
- 物品總大小 = 2.9，下界 $\lceil 2.9 / 1.0 \rceil = 3$ → FFD 達到最佳

### Branch and Bound：剪枝即淘汰

當需要精確最佳解時可採 Branch and Bound：

1. 以某種順序展開「物品如何分配到箱子」的決策樹
2. 對每個節點計算下界（例如 $\lceil \sum s_i / C \rceil$ 加上目前已開箱數）
3. 若下界 $\geq$ 目前已知最佳解 → **整個子樹剪掉**，不再展開

被剪掉的子樹在淘汰與搜尋法的觀點下，等同於「確認該區域不可能含有解」的淘汰動作。

## 結論

淘汰與搜尋法是一種以「縮小問題規模」為核心的解題框架，適用條件：

- 存在一個能在低成本內判斷「哪些資料必然不含答案」的測試或比較
- 每一輪能淘汰固定比例的資料

不同問題對應不同的淘汰策略：偽幣靠天平的三值結果、晶片靠互測配對、二元搜尋靠有序性、第 $k$ 小元素靠 pivot 分割、Bin Packing 的 Branch and Bound 靠下界估算。共同點只有一個——**被淘汰的部分不再回頭**。
