## 淘汰與搜尋法概論

淘汰與搜尋法（Prune and Search）的定義：

> 給定一個搜尋空間，每一步利用某種比較或測試**刪除其中一部分**，只在剩下的部分繼續搜尋，直到找到答案或剩下單一元素。

核心特徵：

- 必須存在一個**判斷規則**，能在低成本（常數時間或線性時間）內決定哪一部分必然不含答案
- 每一輪刪除的比例越大、剩餘規模收斂越快，整體複雜度越好
- 被刪除的部分**不再回頭檢查**

如果每一輪能將規模從 $n$ 縮成 $n / c$（$c > 1$ 為固定常數）：

- 判斷成本為 $O(1)$：$T(n) = T(n/c) + O(1) = O(\log_c n)$
- 判斷成本為 $O(n)$：$T(n) = T(n/c) + O(n) = O(n)$

底數 $c$ 由「一次比較能產生多少種互斥結果」決定。

## 偽幣問題

問題定義：

- 共有 $n$ 枚硬幣，外觀相同
- 其中**恰有一枚是偽幣，已知比真幣輕**
- 工具：一個無砝碼天平
- 目標：找出偽幣，最小化秤量次數

### 演算法步驟

1. 將 $n$ 枚硬幣分成三組，大小盡量相等：左組、右組、未上秤組
2. 將左組與右組放上天平比較
3. 根據結果縮小搜尋範圍：
   - 左輕：偽幣在左組
   - 右輕：偽幣在右組
   - 平衡：偽幣在未上秤組
4. 對被選中的那一組遞迴重複步驟 1–3，直到剩下一枚

### 例題：9 枚硬幣

設 9 枚硬幣編號 0–8，偽幣為 5。

| 步驟 | 左盤 | 右盤 | 未上秤 | 結果 | 候選 |
| :---: | :---: | :---: | :---: | :--- | :--- |
| 1 | `{0,1,2}` | `{3,4,5}` | `{6,7,8}` | 右盤輕 | `{3,4,5}` |
| 2 | `{3}` | `{4}` | `{5}` | 平衡 | `{5}` |

兩次秤量找出偽幣。

### 為什麼分三組：底數的選擇

天平一次比較可以產生**三種互斥結果**（左輕、右輕、平衡），這代表一次比較能將搜尋空間切成三等分，每次只保留其中一份：

$$T(n) = T(n/3) + O(1) = O(\log_3 n)$$

若改用只能回答「是 / 否」兩種結果的工具（例如真假鈔辨識筆，每次只能告訴你「這枚是不是偽幣」），則一次比較只能切成兩等分：

$$T(n) = T(n/2) + O(1) = O(\log_2 n)$$

通則：

- 比較工具能產生 $k$ 種互斥結果 → 每次淘汰可將搜尋空間縮為 $1/k$
- 時間複雜度為 $O(\log_k n)$
- 底數 $k$ 由**比較工具的解析度**決定，不是隨意選擇

| 比較工具 | 一次結果數 | 每輪保留 | 複雜度 |
| :--- | :---: | :---: | :---: |
| 真假辨識筆（是/否） | 2 | $1/2$ | $O(\log_2 n)$ |
| 天平（左/右/平衡） | 3 | $1/3$ | $O(\log_3 n)$ |

### 例題：方向未知的偽幣（4 枚未知 + 參考真幣，兩次秤量）

**問題變體**：有兩組各 4 枚硬幣

- $S_1 = \{A, B, C, D\}$：其中**恰有一枚是偽幣**，但**不知道偏輕或偏重**
- $S_2 = \{T_1, T_2, T_3, T_4\}$：已知全部是真幣，可作為**參考**

用天平**兩次**，找出偽幣，並判斷它是比真幣輕還是重。

**資訊量檢查**：4 枚 $\times$ 2 方向 = 8 種可能情況；2 次秤量 $\times$ 3 種結果 = 9 種結果路徑。$8 \leq 9$，資訊量足夠，可行。

#### 策略：先縮到「1 枚或 3 枚」，再用方向判定

**第一次秤量 $W_1$**：$\{A, B, C\}$ 放左盤，$\{T_1, T_2, T_3\}$ 放右盤

根據 $W_1$ 的結果分成三支：

| $W_1$ 結果 | 推論 | 偽幣候選 | 方向 |
| :--- | :--- | :--- | :--- |
| 平衡 | $A, B, C$ 都是真幣 | $\{D\}$ | 未知 |
| 左重 | $A, B, C$ 中有一枚**重**偽幣 | $\{A, B, C\}$ | 重 |
| 左輕 | $A, B, C$ 中有一枚**輕**偽幣 | $\{A, B, C\}$ | 輕 |

**第二次秤量 $W_2$**：根據 $W_1$ 的結果再設計

```text
若 W1 平衡：
    W2: D 對 T1
    W2 左重 → D 是「重」偽幣
    W2 左輕 → D 是「輕」偽幣
    （平衡不可能——因為保證存在偽幣）

若 W1 左重：     // A, B, C 中有一枚重偽幣
    W2: A 對 B
    W2 左重  → A 是重偽幣
    W2 左輕  → B 是重偽幣
    W2 平衡  → C 是重偽幣

若 W1 左輕：     // A, B, C 中有一枚輕偽幣
    W2: A 對 B
    W2 左重  → B 是輕偽幣   // A 正常、B 輕 → 右盤較輕 → 左盤較重
    W2 左輕  → A 是輕偽幣
    W2 平衡  → C 是輕偽幣
```

#### Pseudocode

```text
FindFake(A, B, C, D, T):                     // T 為已知真幣 (可共用)
    r1 := weigh({A, B, C}, {T, T, T})        // W1

    if r1 == balanced then
        r2 := weigh({D}, {T})                // W2
        if r2 == heavier then return (D, heavy)
        else                return (D, light)

    else if r1 == left-heavier then            // {A,B,C} 有一枚重偽幣
        r2 := weigh({A}, {B})                // W2
        if r2 == left-heavier   then return (A, heavy)
        else if r2 == right-heavier then return (B, heavy)
        else                         return (C, heavy)

    else                                      // r1 == left-lighter；有一枚輕偽幣
        r2 := weigh({A}, {B})                // W2
        if r2 == left-heavier   then return (B, light)
        else if r2 == right-heavier then return (A, light)
        else                         return (C, light)
```

#### 正確性驗證（8 種情況全覆蓋）

| 偽幣 | 方向 | $W_1$ 結果 | $W_2$ 秤量 | $W_2$ 結果 | 演算法判定 | ✓ |
| :---: | :---: | :---: | :--- | :---: | :--- | :---: |
| A | 重 | 左重 | A vs B | 左重 | A 重 | ✓ |
| A | 輕 | 左輕 | A vs B | 左輕 | A 輕 | ✓ |
| B | 重 | 左重 | A vs B | 右重 | B 重 | ✓ |
| B | 輕 | 左輕 | A vs B | 左重 | B 輕 | ✓ |
| C | 重 | 左重 | A vs B | 平衡 | C 重 | ✓ |
| C | 輕 | 左輕 | A vs B | 平衡 | C 輕 | ✓ |
| D | 重 | 平衡 | D vs T | 左重 | D 重 | ✓ |
| D | 輕 | 平衡 | D vs T | 左輕 | D 輕 | ✓ |

8 種情況全部正確判定 → 兩次秤量足夠。

:::tip
關鍵技巧是**用參考真幣壓住不確定性**：第一次把 3 枚未知放左盤、3 枚真幣放右盤，利用天平的三值結果（平衡 / 左重 / 左輕）一次就把「偽幣身分」與「偏向」耦合起來——平衡 $\Rightarrow$ 偽幣是第 4 枚（方向待查）；不平衡 $\Rightarrow$ 方向已定、只剩 3 枚嫌疑犯。第二次秤量剛好各處理 3 種結果，資訊量完美耗盡。
:::

:::warning
**若沒有參考真幣**，4 枚未知硬幣（方向未知）**無法**用兩次秤量同時找出偽幣與判斷輕重。因為任何單純用未知硬幣互比的第一次秤量，都會得到 8 種可能映射到 3 種結果（必有一支結果含超過 3 種情況），第二次秤量無法再把 3 種以上情況分開。這也是為什麼原始 12 硬幣問題（無參考真幣）需要 **3** 次秤量。
:::

## 晶片測試問題

問題定義：

- 共有 $n$ 個晶片，其中好晶片的數量嚴格大於 $n / 2$
- 每次測試取兩個晶片互測，回報結果如下：
  - 兩好：雙方都回報「對方是好的」
  - 一好一壞：好的誠實回報；壞的回報任意
  - 兩壞：雙方回報任意
- 目標：以最少測試次數找出**任一**好晶片

### 演算法步驟

1. 將晶片兩兩配對互測
2. 對每一對的回報：
   - 兩者皆回報「對方好」→ **保留其中一個**，另一個丟棄
   - 任一回報「對方壞」→ **兩個都丟棄**
3. 若 $n$ 為奇數，落單那一片暫時保留
4. 對保留集合遞迴執行步驟 1–3，直到剩下一片

### 正確性簡證

關鍵觀察：在「兩者都回報好」的配對中，兩片必為**同一類**（同好或同壞），不可能一好一壞——因為若一好一壞，好的會誠實回報壞的為壞。

設原本好晶片數為 $g > n/2$，壞晶片為 $b < n/2$。一輪過後：

- 「兩好」配對：保留一片好，丟一片好 → 好晶片數量減半
- 「兩壞」配對：保留一片壞或兩片皆丟（依結果）→ 壞晶片至少減半
- 「一好一壞」配對：必為「對方壞」，兩片都丟 → 各丟掉一好一壞

過濾後好晶片數量仍嚴格大於壞晶片數量，遞迴條件保持成立。

### 複雜度

每輪規模約減半，比較成本 $O(n)$：

$$T(n) = T(n/2) + O(n) = O(n)$$

線性測試次數即可找到一片好晶片。

## 二元搜尋

問題定義：

- 已排序陣列 $a[0..n-1]$
- 給定目標值 $x$
- 找出 $x$ 在陣列中的索引，或回報不存在

### 演算法步驟

以 $\text{lo}$、$\text{hi}$ 表示目前搜尋範圍的左右**索引**，$\text{mid}$ 為中間索引。

1. 設 $\text{lo} = 0$，$\text{hi} = n - 1$
2. 取中點 $\text{mid} = \lfloor (\text{lo} + \text{hi}) / 2 \rfloor$
3. 比較 $a[\text{mid}]$ 與 $x$：
   - 相等：回傳 $\text{mid}$
   - $a[\text{mid}] < x$：$x$ 必在右半 → $\text{lo} \leftarrow \text{mid} + 1$
   - $a[\text{mid}] > x$：$x$ 必在左半 → $\text{hi} \leftarrow \text{mid} - 1$
4. 重複直到 $\text{lo} > \text{hi}$ 或找到 $x$

### 例題

陣列 $[3, 7, 12, 18, 25, 31, 42, 56, 64, 78, 85, 91]$，搜尋 $x = 56$。

| 步驟 | lo（索引） | hi（索引） | mid（索引） | $a[\text{mid}]$ | 動作 |
| :---: | :---: | :---: | :---: | :---: | :--- |
| 1 | 0 | 11 | 5 | 31 | $31 < 56$，淘汰左半 |
| 2 | 6 | 11 | 8 | 64 | $64 > 56$，淘汰右半 |
| 3 | 6 | 7 | 6 | 42 | $42 < 56$，淘汰左半 |
| 4 | 7 | 7 | 7 | 56 | 命中 |

### 例題：1-indexed、上取整、目標不存在

有些課本採 1-indexed 並以 $\text{middle} = \lceil (\text{left} + \text{right}) / 2 \rceil$ 取中點（右偏中點）。邏輯與上面的 0-indexed + 下取整完全一致，差別只在索引從 1 起算、偶數長度時中點落在右半起點。

陣列 $a[1..18] = [1, 2, 4, 6, 8, 10, 12, 13, 16, 18, 20, 28, 30, 35, 36, 41, 44, 49]$，搜尋 $x = 33$。

| 回合 | left | right | $\text{middle} = \lceil (l+r)/2 \rceil$ | $a[\text{middle}]$ | 比較 | 動作 |
| :---: | :---: | :---: | :---: | :---: | :---: | :--- |
| 1 | 1 | 18 | $\lceil 9.5 \rceil = 10$ | 18 | $18 < 33$ | 淘汰左半，`left ← 11` |
| 2 | 11 | 18 | $\lceil 14.5 \rceil = 15$ | 36 | $36 > 33$ | 淘汰右半，`right ← 14` |
| 3 | 11 | 14 | $\lceil 12.5 \rceil = 13$ | 30 | $30 < 33$ | 淘汰左半，`left ← 14` |
| 4 | 14 | 14 | 14 | 35 | $35 > 33$ | 淘汰右半，`right ← 13` |

回合 4 結束後 `left = 14 > right = 13`，跳出迴圈——$x = 33$ **不在陣列中**。

:::info
$x = 33$ 夾在 $a[13] = 30$ 與 $a[14] = 35$ 之間，陣列裡沒有任何一格等於 33。二元搜尋碰到這種情況會把 `left` 與 `right` 推到**交錯**（`left > right`）才結束，收斂在相鄰兩格中間。
:::

### 複雜度

每輪比較一次（$O(1)$），規模減半：

$$T(n) = T(n/2) + O(1) = O(\log n)$$

```python
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

## 選出第 k 小的元素

問題定義：

- 給定 $n$ 個未排序的元素
- 找出第 $k$ 小的元素（$1 \leq k \leq n$）

核心想法：選一個 pivot 將陣列分成「小於 pivot」「等於 pivot」「大於 pivot」三堆，根據第 $k$ 小落在哪一堆，**只遞迴其中一堆**，其餘丟棄。

兩種變體差在 pivot 怎麼挑——一個保證最差情況、一個追求實務期望。

### Median of Medians（BFPRT）

由 Blum、Floyd、Pratt、Rivest、Tarjan 於 1973 年提出，最差情況也能做到 $O(n)$。

#### 步驟

1. 將 $n$ 個元素分成 $\lceil n/5 \rceil$ 組，每組 5 個（最後一組可能不滿）
2. 每組內排序，取中位數，得到 $\lceil n/5 \rceil$ 個中位數
3. **遞迴呼叫 Median of Medians**，找出這些中位數的中位數 $m$
4. 以 $m$ 為 pivot 分割原陣列為 $L$（$< m$）、$E$（$= m$）、$G$（$> m$）
5. 根據 $k$ 的位置：
   - $k \leq |L|$：在 $L$ 中遞迴找第 $k$ 小
   - $|L| < k \leq |L| + |E|$：答案就是 $m$
   - 否則：在 $G$ 中遞迴找第 $(k - |L| - |E|)$ 小

#### 手把手走一遍：25 個數字找第 8 小

起始陣列：

```
[17, 3, 25, 8, 41,  12, 36, 5, 29, 14,  22, 7, 33, 19, 46,  2, 38, 11, 27, 6,  43, 15, 31, 9, 21]
```

目標：$k = 8$，找第 8 小。

**第一輪：25 個元素**

分 5 組，各自排序，取中位數：

| 組 | 原始 | 排序後 | 中位數 |
| :---: | :--- | :--- | :---: |
| G1 | 17, 3, 25, 8, 41 | 3, 8, **17**, 25, 41 | 17 |
| G2 | 12, 36, 5, 29, 14 | 5, 12, **14**, 29, 36 | 14 |
| G3 | 22, 7, 33, 19, 46 | 7, 19, **22**, 33, 46 | 22 |
| G4 | 2, 38, 11, 27, 6 | 2, 6, **11**, 27, 38 | 11 |
| G5 | 43, 15, 31, 9, 21 | 9, 15, **21**, 31, 43 | 21 |

5 個中位數 $\{17, 14, 22, 11, 21\}$ 排序後 $\{11, 14, 17, 21, 22\}$，**中位數 = 17** → pivot。

以 $m = 17$ 分堆：

- $L$（$< 17$）：`[3, 8, 12, 5, 14, 7, 2, 11, 6, 15, 9]`，共 11 個
- $E$（$= 17$）：`[17]`，共 1 個
- $G$（$> 17$）：`[25, 41, 36, 29, 22, 33, 19, 46, 38, 27, 43, 31, 21]`，共 13 個

$k = 8 \leq |L| = 11$ → 答案在 $L$ 中，進第二輪找第 8 小。

**第二輪：$L$ 共 11 個元素**

$L = [3, 8, 12, 5, 14, 7, 2, 11, 6, 15, 9]$

分組（最後一組不滿 5 個）：

| 組 | 原始 | 排序後 | 中位數 |
| :---: | :--- | :--- | :---: |
| G1 | 3, 8, 12, 5, 14 | 3, 5, **8**, 12, 14 | 8 |
| G2 | 7, 2, 11, 6, 15 | 2, 6, **7**, 11, 15 | 7 |
| G3 | 9 | **9** | 9 |

中位數 $\{8, 7, 9\}$ 排序後 $\{7, 8, 9\}$，**中位數 = 8** → pivot。

以 $m = 8$ 分堆：

- $L'$（$< 8$）：`[3, 5, 7, 2, 6]`，5 個
- $E'$（$= 8$）：`[8]`，1 個
- $G'$（$> 8$）：`[12, 14, 11, 15, 9]`，5 個

$k = 8 > |L'| + |E'| = 6$ → 答案在 $G'$ 中，新的 $k = 8 - 5 - 1 = 2$，進第三輪。

**第三輪：$G'$ 共 5 個元素**

$G' = [12, 14, 11, 15, 9]$，只需一組：

| 組 | 排序後 | 中位數 |
| :---: | :--- | :---: |
| G1 | 9, 11, **12**, 14, 15 | 12 |

pivot $m = 12$，分堆：

- $L''$（$< 12$）：`[11, 9]`，2 個
- $E''$（$= 12$）：`[12]`，1 個
- $G''$（$> 12$）：`[14, 15]`，2 個

$k = 2 \leq |L''| = 2$ → 在 $L''$ 中找第 2 小，進第四輪。

**第四輪：$L''$ 共 2 個元素**

$L'' = [11, 9]$，排序後 $[9, 11]$，第 2 小 = **11**。

**驗證**：原陣列排序後為 $2, 3, 5, 6, 7, 8, 9, \mathbf{11}, 12, 14, \ldots$，第 8 位確實是 11 ✓

#### 例題：27 個數字找第 16 小

陣列 $a[1..27] = [7, 3, 6, 8, 1, 10, 4, 3, 8, 5, 7, 10, 0, 10, 2, 1, 4, 4, 13, 5, 0, 8, 9, 9, 2, 6, 1]$，目標 $k = 16$。

**(a) 分組求中位數**

27 個元素分成 6 組（前 5 組各 5 個、最後一組 2 個）：

| 組 | 原始 | 排序後 | 中位數 |
| :---: | :--- | :--- | :---: |
| G1 | 7, 3, 6, 8, 1 | 1, 3, **6**, 7, 8 | 6 |
| G2 | 10, 4, 3, 8, 5 | 3, 4, **5**, 8, 10 | 5 |
| G3 | 7, 10, 0, 10, 2 | 0, 2, **7**, 10, 10 | 7 |
| G4 | 1, 4, 4, 13, 5 | 1, 4, **4**, 5, 13 | 4 |
| G5 | 0, 8, 9, 9, 2 | 0, 2, **8**, 9, 9 | 8 |
| G6 | 6, 1 | 1, **6** | 6 |

六個中位數 $M = \{6, 5, 7, 4, 8, 6\}$。

:::warning
常見錯誤：G4 的中位數誤寫成 5、G5 誤寫成 6。關鍵是**先排序**再取正中間那個。五個元素的中位數在排序後第 3 位，不是「原始順序第 3 個」。
:::

**(b) 取中位數的中位數（MoM）作為 pivot**

把 $M = \{6, 5, 7, 4, 8, 6\}$ 排序得 $\{4, 5, 6, 6, 7, 8\}$，中位數 = **6** → pivot $m = 6$。

**(c) 以 $m = 6$ 分堆**

掃過 $a[1..27]$，依值分到三堆：

| 堆 | 內容 | 大小 |
| :---: | :--- | :---: |
| $L$（$< 6$） | 3, 1, 4, 3, 5, 0, 2, 1, 4, 4, 5, 0, 2, 1 | 14 |
| $E$（$= 6$） | 6, 6 | 2 |
| $G$（$> 6$） | 7, 8, 10, 8, 7, 10, 10, 13, 8, 9, 9 | 11 |

檢查 $k = 16$ 落在哪裡：

- $\lvert L \rvert = 14$，$\lvert L \rvert + \lvert E \rvert = 16$
- $14 < 16 \leq 16$ → $k$ 落在 $E$ 中 → 答案就是 pivot = **6**

**驗證**：$a[1..27]$ 排序後為

$$0, 0, 1, 1, 1, 2, 2, 3, 3, 4, 4, 4, 5, 5, \mathbf{6}, \mathbf{6}, 7, 7, 8, 8, 8, 9, 9, 10, 10, 10, 13$$

第 16 位確實是 **6** ✓。本題 $k$ 剛好落在 $E$ 區，不必再遞迴第二輪。

#### 為什麼分 5 個一組

| 組大小 | 每輪最多剩下 | 是否線性 |
| :---: | :---: | :---: |
| 3 | $2n/3$ | 否（$1/3 + 2/3 = 1$，不收斂） |
| 5 | $7n/10$ | 是（$1/5 + 7/10 = 9/10 < 1$） |
| 7 | $5n/7$ | 是，但常數更大 |

- 組大小 3 不收斂 → 無法線性
- 組大小 5 是「最小可線性」的選擇，分組成本最低
- 組大小 7 常數變大、不划算

「5」是最優平衡點。

#### 複雜度

- 可證明每輪淘汰至少 $3n/10$ 個元素，遞迴規模最多剩 $7n/10$
- 最差情況：$O(n)$

### Randomized Select

#### 步驟

1. 從陣列中**隨機**選一個元素作為 pivot $p$
2. 分割為 $L$（$< p$）、$E$（$= p$）、$G$（$> p$）
3. 根據 $k$ 的位置遞迴 $L$ 或 $G$，或直接回傳 $p$

#### 手把手走一遍：同一組數字找第 8 小

用同一組 25 個數字，$k = 8$，但 pivot 隨機挑。以下是一次執行範例（實際每次執行會不同）。

**第一輪：25 個元素**

原陣列：`[17, 3, 25, 8, 41, 12, 36, 5, 29, 14, 22, 7, 33, 19, 46, 2, 38, 11, 27, 6, 43, 15, 31, 9, 21]`

隨機抽到 pivot = **21**。分堆：

- $L$（$< 21$）：`[17, 3, 8, 12, 5, 14, 7, 19, 2, 11, 6, 15, 9]`，13 個
- $E$（$= 21$）：`[21]`，1 個
- $G$（$> 21$）：`[25, 41, 36, 29, 22, 33, 46, 38, 27, 43, 31]`，11 個

$k = 8 \leq |L| = 13$ → 進 $L$ 找第 8 小。

**第二輪：$L$ 共 13 個元素**

$L = [17, 3, 8, 12, 5, 14, 7, 19, 2, 11, 6, 15, 9]$

隨機抽到 pivot = **9**。分堆：

- $L'$（$< 9$）：`[3, 8, 5, 7, 2, 6]`，6 個
- $E'$（$= 9$）：`[9]`，1 個
- $G'$（$> 9$）：`[17, 12, 14, 19, 11, 15]`，6 個

$k = 8 > |L'| + |E'| = 7$ → 進 $G'$ 找第 $8 - 6 - 1 = 1$ 小。

**第三輪：$G'$ 共 6 個元素**

$G' = [17, 12, 14, 19, 11, 15]$

隨機抽到 pivot = **12**。分堆：

- $L''$（$< 12$）：`[11]`，1 個
- $E''$（$= 12$）：`[12]`，1 個
- $G''$（$> 12$）：`[17, 14, 19, 15]`，4 個

$k = 1 \leq |L''| = 1$ → 進 $L''$。

**第四輪：$L'' = [11]$**

單一元素，直接回傳 **11**。

答案 = 11 ✓（與 BFPRT 結果一致）

差別：BFPRT 每輪 pivot 固定為中位數的中位數、成本較高但穩；Randomized Select 直接抽一個、多數時候也很快，但若連續抽到極端值最差會退化成 $O(n^2)$。

#### 程式碼

```python
import random

def randomized_select(arr, k):
    if len(arr) == 1:
        return arr[0]
    pivot = random.choice(arr)
    L = [x for x in arr if x < pivot]
    E = [x for x in arr if x == pivot]
    G = [x for x in arr if x > pivot]
    if k <= len(L):
        return randomized_select(L, k)
    elif k <= len(L) + len(E):
        return pivot
    else:
        return randomized_select(G, k - len(L) - len(E))
```

#### 複雜度

- 期望：$O(n)$
- 最差（連續抽到最極端元素）：$O(n^2)$，機率 $1/n!$，實務幾乎不會出現

### 兩種變體比較

| | Median of Medians | Randomized Select |
| :---: | :---: | :---: |
| pivot 選擇 | 中位數的中位數 | 隨機 |
| 期望複雜度 | $O(n)$ | $O(n)$ |
| 最差複雜度 | $O(n)$ | $O(n^2)$ |
| 常數 | 大 | 小 |
| 實務使用 | 少 | 多 |

## 打包問題（Bin Packing）

問題定義：

- 給定 $n$ 個物品，大小分別為 $s_1, s_2, \ldots, s_n$，皆滿足 $0 < s_i \leq C$
- 給定箱子容量 $C$
- 目標：使用**最少數量**的箱子裝下所有物品

Bin Packing 為 **NP-Hard** 問題，無已知多項式時間最佳解。實務上以**近似演算法**為主，淘汰與搜尋法不是 Bin Packing 的主力解法，但其中的**剪枝精神**與淘汰與搜尋法相通——分支限界（Branch and Bound）會剪掉「已知不可能優於目前最佳解」的子樹。

### WG 與 WE：浪費的兩種來源

要分析近似演算法的好壞，需要量化「裝箱不夠緊」造成的浪費：

- **WG（Wasted Group / Wasted Bin）**：一個箱子被關閉（不再放入物品）時，**剩餘的未使用容量**。例如容量 $C = 10$，箱子裝了 7 後關閉，則該箱貢獻 $\text{WG} = 3$。
- **WE（Wasted Element）**：一個物品因為**現有所有箱子都塞不下**，被迫開新箱來容納。這個物品本身就是一次「淘汰失敗」事件——它被現有箱子全部拒絕。

兩者關係：

- 總空間使用 = 物品總大小 + 所有 WG 的總和
- 開的箱子數 $\geq \lceil$ 物品總大小 $/ C \rceil$，超出的部分由 WG 累積決定
- 每出現一個 WE，至少多開一個箱子，並可能在原本箱子留下新的 WG

### 不同近似策略對 WG / WE 的影響

| 策略 | 機制 | WG 趨勢 | WE 趨勢 |
| :--- | :--- | :--- | :--- |
| Next Fit | 只看最後一個箱子，塞不下就開新箱 | 高（前面箱子永遠不再使用） | 高 |
| First Fit | 從第一個箱子掃，第一個塞得下就放 | 中（會回頭利用） | 中 |
| Best Fit | 選擇剩餘空間最少且仍能容納的箱子 | 較低（剩餘空間集中於少數箱） | 中 |
| First Fit Decreasing | 先將物品由大到小排序，再 First Fit | 低（小物件用來填縫） | 低 |

直覺上：先放大的物品，剩下的縫隙才有機會被小物品填滿——這就是 FFD 比 FF 更好的原因。

### 近似比結論

經典結果（證明略，皆是透過 WG / WE 的上界推出）：

- **First Fit**：$A \leq 1.7 \cdot \text{OPT} + 2$
- **First Fit Decreasing**：$A \leq \frac{11}{9} \cdot \text{OPT} + \frac{6}{9}$

直覺理解：WG 累積越少、WE 越少發生，近似比越好；FFD 先排大物品，留下的縫隙最容易被小物品填滿，因此上界最緊。

### 例題：First Fit Decreasing

物品：$[0.5, 0.7, 0.5, 0.2, 0.4, 0.2, 0.3, 0.1]$，箱容量 $C = 1.0$

排序後（由大到小）：$[0.7, 0.5, 0.5, 0.4, 0.3, 0.2, 0.2, 0.1]$

| 步驟 | 物品 | 動作 | 箱況（剩餘容量） | WG 變化 | WE |
| :---: | :---: | :--- | :--- | :---: | :---: |
| 1 | 0.7 | 開 Bin1 | Bin1: 0.3 | — | — |
| 2 | 0.5 | Bin1 塞不下（0.3），開 Bin2 | Bin1: 0.3, Bin2: 0.5 | — | +1 |
| 3 | 0.5 | Bin1 塞不下，Bin2 剛好 | Bin1: 0.3, Bin2: 0.0 | — | — |
| 4 | 0.4 | Bin1 塞不下，Bin2 塞不下，開 Bin3 | Bin1: 0.3, Bin2: 0.0, Bin3: 0.6 | — | +1 |
| 5 | 0.3 | Bin1 剛好 | Bin1: 0.0, Bin2: 0.0, Bin3: 0.6 | — | — |
| 6 | 0.2 | Bin3 可塞 | Bin3: 0.4 | — | — |
| 7 | 0.2 | Bin3 可塞 | Bin3: 0.2 | — | — |
| 8 | 0.1 | Bin3 可塞 | Bin3: 0.1 | — | — |

最終結果：

- 使用 3 個箱子
- Bin1 WG = 0，Bin2 WG = 0，Bin3 WG = 0.1
- 總 WG = 0.1，總 WE 事件 = 2（步驟 2、4）
- 物品總大小 = 2.9，下界 $\lceil 2.9 / 1.0 \rceil = 3$ → FFD 達到最佳

### Branch and Bound：剪枝即淘汰

當需要精確最佳解時可採 Branch and Bound：

1. 以某種順序展開「物品如何分配到箱子」的決策樹
2. 對每個節點計算下界（例如 $\lceil \sum s_i / C \rceil$ 加上目前已開箱數）
3. 若下界 $\geq$ 目前已知最佳解 → **整個子樹剪掉**，不再展開

被剪掉的子樹在淘汰與搜尋法的觀點下，等同於「確認該區域不可能含有解」的淘汰動作。

## 結論

淘汰與搜尋法是一種以「縮小問題規模」為核心的解題框架，適用條件：

- 存在一個能在低成本內判斷「哪些資料必然不含答案」的測試或比較
- 每一輪能淘汰固定比例的資料

不同問題對應不同的淘汰策略：偽幣靠天平的三值結果、晶片靠互測配對、二元搜尋靠有序性、第 $k$ 小元素靠 pivot 分割、Bin Packing 的 Branch and Bound 靠下界估算。共同點只有一個——**被淘汰的部分不再回頭**。
