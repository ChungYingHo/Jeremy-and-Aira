## 資料在元件之間怎麼流動？

用 Vue 寫到一定程度，一定會碰到一個問題：**元件之間怎麼傳資料？**

一個頁面可能有 Header、Sidebar、Content、Footer，每個又拆成更小的元件。當 A 元件的資料需要給 B 元件用的時候，怎麼辦？

Vue 提供了好幾種方式，但核心觀念只有一個：

> **單向資料流（One-way Data Flow）**

簡單來說就是：資料從父元件往下流到子元件，就像水從上往下流一樣。子元件不能直接改父元件的資料，它只能「通知」父元件說：「欸，我這邊有事情發生了，你自己決定要不要改。」

為什麼要這樣設計？因為如果每個元件都能隨便改別人的資料，當 bug 出現的時候你根本不知道「到底是誰改的」。單向資料流讓資料的來源和流向都很清楚，debug 的時候不會想砸電腦。

好，概念講完了，來看實際怎麼做。

## props — 父傳子

最基本的傳資料方式。父元件透過屬性把資料傳給子元件。

```vue
<!-- 父元件 -->
<script setup>
import { ref } from 'vue'
import UserCard from './UserCard.vue'

const userName = ref('Jeremy')
</script>

<template>
  <UserCard :name="userName" />
</template>
```

```vue
<!-- 子元件 UserCard.vue -->
<script setup>
const props = defineProps({
  name: {
    type: String,
    required: true
  }
})
</script>

<template>
  <div class="card">
    <p>名字：{{ name }}</p>
  </div>
</template>
```

### defineProps 的幾種寫法

Vue 3 提供了兩種定義 props 的方式：

**Runtime 宣告（傳統寫法）：**
```js
const props = defineProps({
  name: String,
  age: {
    type: Number,
    default: 18
  },
  items: {
    type: Array,
    default: () => []  // 物件/陣列的預設值要用函式
  }
})
```

**TypeScript 寫法：**
```ts
const props = defineProps<{
  name: string
  age?: number
  items?: string[]
}>()
```

如果用 TS 寫法又想要預設值，可以搭配 `withDefaults`：
```ts
const props = withDefaults(defineProps<{
  name: string
  age?: number
}>(), {
  age: 18
})
```

### props 是唯讀的！

這很重要。子元件**不能直接修改**父元件傳下來的 props：

```js
// ❌ 這樣會噴警告
props.name = 'Aira'
```

想改的話有兩個選擇：
1. 用 `emit` 通知父元件去改（推薦）
2. 在子元件內複製一份到本地的 `ref`（某些情境適用）

```js
// 複製一份到本地 ref
const localName = ref(props.name)
// 注意：之後 props.name 再變，localName 不會跟著變
```

## emit — 子傳父

子元件不能改 props，但它可以「發射事件」告訴父元件：「嘿，我這邊出事了！」

用生活比喻：你是餐廳服務生（子元件），客人點了餐你不能直接去廚房煮，但你可以按鈴通知廚師（父元件）：「一號桌點了牛排！」

```vue
<!-- 子元件 Counter.vue -->
<script setup>
const emit = defineEmits(['increment', 'decrement'])

function handleClick() {
  emit('increment', 1)  // 第二個參數是要帶給父元件的資料
}
</script>

<template>
  <button @click="handleClick">+1</button>
  <button @click="emit('decrement', 1)">-1</button>
</template>
```

```vue
<!-- 父元件 -->
<script setup>
import { ref } from 'vue'
import Counter from './Counter.vue'

const count = ref(0)

function handleIncrement(n) {
  count.value += n
}

function handleDecrement(n) {
  count.value -= n
}
</script>

<template>
  <p>目前數量：{{ count }}</p>
  <Counter @increment="handleIncrement" @decrement="handleDecrement" />
</template>
```

### TypeScript 的 emit 定義

```ts
const emit = defineEmits<{
  increment: [amount: number]
  decrement: [amount: number]
}>()
```

## provide / inject — 跨層級傳資料

props 和 emit 適合父子之間傳資料。但如果你的元件結構是這樣的：

```
App
└── Layout
    └── Sidebar
        └── UserInfo
            └── Avatar    ← 它需要 App 層的 user 資料
```

難道要每一層都用 props 傳下去嗎？

```
App (user) → Layout (:user) → Sidebar (:user) → UserInfo (:user) → Avatar (:user)
```

這就是所謂的 **props drilling**（道具鑽孔），每一層都要「鑽一個洞」讓資料穿過去，中間那些元件根本不需要用到這個資料，但為了傳遞不得不接收。維護起來超級痛苦。

`provide` / `inject` 就是來解決這個問題的。祖先元件用 `provide` 提供資料，後代元件用 `inject` 直接取用，中間層不用管。

```vue
<!-- 祖先元件 App.vue -->
<script setup>
import { ref, provide } from 'vue'

const user = ref({ name: 'Jeremy', role: 'admin' })

provide('user', user)  // key, value
</script>
```

```vue
<!-- 後代元件 Avatar.vue（不管隔了幾層） -->
<script setup>
import { inject } from 'vue'

const user = inject('user')
// user 是響應式的，因為提供的就是 ref
</script>

<template>
  <img :alt="user?.name" />
  <span>{{ user?.name }}</span>
</template>
```

### provide / inject 的注意事項

1. **只能往下**：inject 只能取到祖先 provide 的資料，不能反過來。
2. **預設值**：如果沒有祖先 provide 對應的 key，可以給預設值：
   ```js
   const user = inject('user', { name: 'Guest' })
   ```
3. **唯讀**：如果你希望注入的資料不被子孫改掉，可以用 `readonly`：
   ```js
   provide('user', readonly(user))
   ```
4. **用 Symbol 當 key**：避免命名衝突，在大型專案裡建議用 Symbol：
   ```js
   // keys.js
   export const USER_KEY = Symbol('user')

   // 提供
   provide(USER_KEY, user)

   // 注入
   const user = inject(USER_KEY)
   ```

### 什麼時候用 provide / inject？

- 跨很多層的資料（theme、locale、user 資訊）
- 跟一組元件共享狀態（比如 `<Form>` 跟它底下的所有 `<Input>` 共享驗證邏輯）
- 不想引入全域狀態管理（Pinia）但又需要跨元件共享

但如果你的狀態需要在**不相關的元件**之間共享（比如 Header 和 Footer），或者狀態邏輯很複雜，那還是用 Pinia 吧。

## v-model — 雙向綁定的語法糖

`v-model` 大家應該都用過，最常見的就是表單：

```vue
<input v-model="name" />
```

但 `v-model` 其實就是 props + emit 的語法糖！上面那行等同於：

```vue
<input :value="name" @input="name = $event.target.value" />
```

### 在自訂元件上用 v-model

這是實務上很常用的技巧。當你封裝一個自訂的 Input 元件時：

```vue
<!-- 父元件 -->
<CustomInput v-model="searchText" />
```

```vue
<!-- CustomInput.vue -->
<script setup>
const model = defineModel()
</script>

<template>
  <input :value="model" @input="model = $event.target.value" />
</template>
```

Vue 3.4 引入了 `defineModel`，讓這件事變得超級簡單。在此之前你需要自己定義 props 和 emit，現在一行搞定。

### 多個 v-model

一個元件可以有多個 `v-model`：

```vue
<!-- 父元件 -->
<UserForm
  v-model:firstName="first"
  v-model:lastName="last"
/>
```

```vue
<!-- UserForm.vue -->
<script setup>
const firstName = defineModel('firstName')
const lastName = defineModel('lastName')
</script>

<template>
  <input v-model="firstName" placeholder="First" />
  <input v-model="lastName" placeholder="Last" />
</template>
```

### v-model 修飾符

Vue 內建了幾個好用的修飾符：

```vue
<!-- .lazy：從 input 事件改成 change 事件（失去焦點才更新） -->
<input v-model.lazy="msg" />

<!-- .number：自動轉成數字 -->
<input v-model.number="age" type="number" />

<!-- .trim：自動去除頭尾空白 -->
<input v-model.trim="name" />
```

## 什麼場景用什麼方式？

| 場景 | 方式 | 說明 |
|---|---|---|
| 父 → 子傳資料 | `props` | 最基本，單向往下 |
| 子 → 父通知事件 | `emit` | 子元件不能改 props，只能發事件 |
| 跨多層傳資料 | `provide / inject` | 避免 props drilling |
| 表單雙向綁定 | `v-model` | props + emit 的語法糖 |
| 全域狀態 | Pinia | 不相關元件間共享，邏輯複雜時用 |

### 決策流程

1. 只是父子之間傳？→ **props + emit**
2. 需要雙向綁定？→ **v-model**
3. 跨好幾層但只限某棵子樹？→ **provide / inject**
4. 整個 app 都要用到？→ **Pinia**（後面的文章會講）

其實資料流的核心觀念真的就是「單向資料流」。理解了這個，其他的方式都只是在不同情境下的工具選擇而已。不管用哪種方式，資料的「源頭」都應該要清楚可追蹤，這樣 debug 的時候才不會一臉茫然。
