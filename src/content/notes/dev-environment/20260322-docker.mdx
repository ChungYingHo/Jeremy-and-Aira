## 容器是什麼

**容器（Container）** 將應用程式及其所有依賴（執行環境、套件、設定檔）打包成一個獨立單元，確保在任何機器上行為一致。

- 解決傳統部署中作業系統、語言版本、系統函式庫不一致導致程式無法執行的問題
- 輕量級隔離——包含程式執行所需的一切，但不需要完整模擬作業系統

## 容器 vs 虛擬機

| | 虛擬機 (VM) | 容器 (Container) |
| :--- | :--- | :--- |
| 隔離層級 | 完整模擬硬體，跑獨立的 OS | 共用 Host OS 核心，只隔離應用程式 |
| 啟動速度 | 分鐘級 | 秒級至毫秒級 |
| 大小 | GB 等級 | MB 等級 |
| 用途 | 需要完全隔離的環境 | 應用程式部署與開發環境 |

虛擬機模擬完整硬體並運行獨立 OS；容器共用 Host OS 核心，僅隔離應用層。

## Docker 的核心概念

### Image（映像檔）

- 容器的唯讀模板，定義容器內容與啟動方式
- 從 **Dockerfile** 建構，或從 [Docker Hub](https://hub.docker.com/) 下載現成 image（如 Node.js、PostgreSQL、Nginx）

### Container（容器）

- Image 的執行實例；同一個 Image 可同時運行多個 Container，彼此隔離
- Container 為暫時性——刪除後資料消失，需透過 **Volume** 實現資料持久化

### Dockerfile

描述如何建構 Image 的腳本，每一行指令在 Image 中新增一層：

```dockerfile
# 從官方 Node.js 20 image 開始
FROM node:20-alpine

# 設定工作目錄
WORKDIR /app

# 複製 package.json 並安裝依賴
COPY package.json .
RUN npm install

# 複製其餘程式碼
COPY . .

# 啟動指令
CMD ["node", "index.js"]
```

### Docker Compose

用於多服務場景（如 Web Server + 資料庫 + Cache）。透過單一 `compose.yml` 定義所有服務，一個指令全部啟動。

```yaml
services:
  web:
    image: nginx
    ports:
      - "80:80"

  db:
    image: postgres:16
    environment:
      POSTGRES_PASSWORD: secret
    volumes:
      - db_data:/var/lib/postgresql/data

volumes:
  db_data:
```

## Docker 解決了什麼

- **開發環境一致性**：`docker compose up` 即可取得統一環境，新人 onboarding 無需逐一安裝工具
- **應用程式隔離**：不同專案各自運行於獨立容器，語言與套件版本互不衝突
- **部署一致性**：本機測試通過的 Image 與 production 環境完全相同，減少環境差異導致的問題

## Registry：Image 的倉庫

**Registry** 是存放 Image 的遠端儲存庫。

- **Docker Hub**：最常用的公開 Registry，大多數官方 image 存放於此
- **GitHub Container Registry (GHCR)**：整合 GitHub，適合開源專案
- **Amazon ECR / Google Artifact Registry / Azure Container Registry**：各雲端平台的私有 Registry

### 推送與拉取流程

以推到 GHCR 為例：

```bash
# 1. 登入（使用 GitHub personal access token）
echo $GHCR_TOKEN | docker login ghcr.io -u <username> --password-stdin

# 2. 把本機 image 打上遠端 tag
docker tag my-app:latest ghcr.io/<username>/my-app:1.0.0

# 3. 推到 Registry
docker push ghcr.io/<username>/my-app:1.0.0

# 4. 在另一台機器拉下來直接跑
docker pull ghcr.io/<username>/my-app:1.0.0
docker run ghcr.io/<username>/my-app:1.0.0
```

- Tag 格式：`<registry>/<namespace>/<image>:<version>`
- 省略 `<registry>` 預設 Docker Hub
- 同一個 image 可以打多個 tag（例如 `1.0.0` 與 `latest`）同時推送

## 和雲端的關係

Docker 在本機或伺服器上運行容器。大規模部署、自動擴縮容、高可用性場景需要**容器編排**工具，主流為 **Kubernetes（K8s）**。

各雲端平台提供託管容器服務：

- **AWS ECS / EKS**：Amazon 的容器服務
- **Google Cloud Run / GKE**：Google 的容器服務
- **Azure Container Apps / AKS**：Microsoft 的容器服務

這些服務皆以 Docker Image 為基礎，推送 Image 後由平台處理部署與維運。
